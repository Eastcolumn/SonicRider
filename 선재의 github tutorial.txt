git init
git clone ~~.git

git
Local, Remote
내 컴퓨터, 깃허브

git status

로컬 깃에 아직 안 추가한거랑, 추가 된것들 확인이 가능

git add . : 모든 파일
git add 파일명

로컬 깃에 스테이징 ( 추가는 아직 X )

git commit -m "메시지" 

로컬 깃에 커밋을 기록

git push -u origin master

Remote의 master 브랜치로 로컬 커밋을 전달 -> 레모트 커밋 생성

rm -rf 폴더, 파일을 강제로 삭제
( r : recursive, f : force )

git branch 브랜치명 : 로컬 깃에 새로운 브랜치를 생성

git checkout 브랜치명 : 브랜치 이동

git push -u origin 브랜치명 : 레모트 깃의 브랜치로 커밋 생성

develop -> git merge master

중복되는 부분 : Conflict
선택을 해야함. 어느 것을 따를 것인지

별다른 중복이 없으면, 알아서 합쳐줌.

git log : 커밋 기록 확인
